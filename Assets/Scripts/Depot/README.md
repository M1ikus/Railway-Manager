# Depot — przewodnik

> Dokumentacja modułu `Assets/Scripts/Depot/`.

## Asmdef + warstwa

**`RailwayManager.Depot`**, refs: `Core, Fleet, SharedUI, MainMenu` (+ TMP + InputSystem).
**`rootNamespace` jest pusty w `.asmdef`** — pliki same deklarują namespace.
Większość siedzi w `DepotSystem`, podsystemy w `DepotSystem.Furniture`, `DepotSystem.Modernization`,
`DepotSystem.OutdoorEquipment`. Schemas w `RailwayManager.Depot.Schemas`.

**Reguła:** Depot **NIE może referować Map ani Timetable**. Cykl Depot↔Map został zerwany
przez przeniesienie integracji do Map (Map widzi Depot, nie odwrotnie). Cross-system
komunikacja w stronę Map idzie przez Core (`VehicleLocationService`, `UIIntents`).

**Scena:** `Depot.unity` — pierwsza scena gry po MainMenu, edytor zajezdni 3D.

## Cross-system glue

- **Map↔Depot M9c handshake**: `VehicleLocationService.SetExitingDepot/SetEnteringDepot` (Core), `DepotMovementSimulator.SpawnConsistAtEntry`, `OnConsistEnteredDepot` event.
- **Personnel↔Depot M-Furniture bridge**: `FurnitureAssignmentService` (Personnel asmdef) konsumuje `FurniturePlacer.OnPlacementStateChanged`. `ServicePitFurnitureBridge` (Maintenance asmdef po splicie 2026-05-15, wcześniej Timetable, bo Depot nie może referować Maintenance/Timetable) — asymetria wynika z hierarchii asmdef.
- **Personnel walk**: `EmployeeWalkPathfinder` w Personnel asmdef konsumuje **`PathGraph`** z Depot (BFS + polyline waypoints).
- **Timetable↔Depot M-Modernization**: `WorkshopManager.SlotSpeedMultiplierHook` (z Personnel), `DispatchActionService` integration z TrafficController, manual dispatch fallback.
- **UIIntents → Depot**: subskrybenci `OpenWorkshopsPanel`/`OpenPartsPanel`/`OpenFinancesPanel` siedzą w innych asmdef ale otwierają panele zarejestrowane w Depot scene.
- **Save/Load**: `DepotSavable` (w SaveLoad asmdef) reflection-discovers `ISavable` w Depot — schema v3→v5 (M-Furniture + M-Modernization).

## Gotchas

- **`DepotSystem` namespace ≠ `RailwayManager.Depot.Schemas` namespace** — Schemas wymaga full namespace dla `using`. Pomyłka w `using` to częsty błąd.
- **`WallSegment` jest top-level w `DepotSystem`** (nie nested w `WallBuildingSystem`). Anti-pattern z root CLAUDE.md.
- **Depot nie może `using RailwayManager.Timetable`** ani `RailwayManager.Map` — bridges muszą siedzieć w wyższych asmdef i konsumować Depot events (subskrypcja na `FurniturePlacer.OnPlacementStateChanged` etc.).
- **`TrackGraph` jest `MonoBehaviour` partial × 6** — modyfikacje muszą leżeć w odpowiednim partial (Nodes/Edges/Tracks/Pathfinding/Switches). NIE wrzucaj wszystkiego do roota.
- **TD-031 occupancy = interwały, nie binarne (2026-06-08).** Źródło prawdy to `DepotTrackData.Occupants` (`List<TrackOccupant>`, footprinty [Front,Rear] track-local). Pola `IsOccupied`/`OccupyingConsistId`/`OccupyingVehicleIds` to **pochodny legacy-mirror** (pierwszy occupant) — czytaj je tylko w display/UI, NIE do logiki ruchu. Nowy kod: `IsRangeFreeFor`/`SetOccupantInterval`/`GetOccupants`/`RemoveConsistEverywhere`. `currentDistanceM` w `DepotMoveTask` to **ŚRODEK** składu (footprint = center ± L/2) — nie nos. Cap dojazdu odejmuje `L/2 + ContactGapM`.
- **`generateDefaultLayout` w `DepotManager`** = true triggeruje `GenerateDefaultLayout()` w Start. Test sceny mogą chcieć false. Permanentne tory zewnętrzne (`GenerateExternalTracks`) idą zawsze, niezależnie od flag.
- **`PathGraph` jest osobny od `TrackGraph`** — bez catenary/blade/turnout. Workers chodzą po PathGraph; pojazdy po TrackGraph. NIE mieszaj — to inne API.
- **`FleetPanelUI` siedzi w Depot/UI/, nie w Fleet asmdef** — bo wymaga 3D preview (PaintPreview3D, orbit cam). 13 partials w sumie (legacy single-model configurator + seat-layout partials usuniete M-UIPolish 2026-06-18).
- **`Undo` stack jest globalny per UndoManager singleton** — każde destrukcyjne action MUSI rejestrować `IUndoCommand` przed zmianą (BUG-pattern: zapominanie undo zostawia nieodwracalny state w MoveInstance — patrz root CLAUDE.md anti-skróty).
- **Camera input gating**: `DepotOrbitCamera.Update` sprawdza `SceneController.TimetablePopupOpen` i `FullscreenOverlayOpen` — gdy dodajesz nowy popup, dorzuć flagę i sprawdzanie.
- **Używaj `DepotServices.Get<T>()` zamiast `FindAnyObjectByType<T>` w runtime** (2026-05-13). Auto-invalidate na scene unload, ~10× faster post-EA. Jedyne wyjątki gdzie zostaje `FindAnyObjectByType`: AutoSpawn singleton ensure (`if (FindAnyObjectByType<X>() != null) return;` w `[RuntimeInitializeOnLoadMethod]`) + Unity built-in `EventSystem` lookup w `DepotUIManager.BuildCanvas`. Smoke tests też zostają.
- **Nowe tool state machines: użyj `ToolModeGate` composition helper** (2026-05-13). Zamiast `void Update() { if (CurrentTool != X) return; ... }` + defensive Cancel*/Hide*/Clear* per frame, w Start stwórz `_toolGate = new ToolModeGate(this, m => m == ToolMode.X, OnToolDeactivated); _toolGate.Start();` i Stop w OnDestroy. Unity nie wywołuje Update/LateUpdate gdy enabled=false. Wzorzec użycia: 8 klas (ElectrificationStateMachine, HallActionStateMachine, OutdoorEquipmentPlacer, TrackBuildStateMachine, PathBuildStateMachine, WallBuildingSystem, FurnitureSelector, DepotConsistSelectionHandler).
- **Nowe procedural UI: użyj `DepotUIPrimitives` helpers** (2026-05-13). `DepotUIPanelPrimitives.CreateTMP/CreateButtonColors/CreateOptionButton/...` — patrz `DepotUIPrimitives.cs`. Wcześniej te helpers były nested w `MainTabBarUI.cs`, mimo że używane w 8 plikach UI.
- **MonoBehaviour z partial × N**: przed dodaniem `void OnDestroy()`/`Awake`/`Start` w nowym partialu **grep cały plik chain** czy callback już nie istnieje (lekcja z CS0111 w ElectrificationStateMachine, 2026-05-13).
- **Billing rozjazdów = pełne lustro (TD-035, 2026-06-10)**: place robi swap geometrii (refund usuniętego chain + charge nowych segmentów) + charge mechanizmu; remove DOKŁADNIE odwrotnie (refund members+pre/post + restore CHARGED + refund mechanizmu) → każdy cykl place→remove i undo = net-zero. NIE dodawaj suppress/warunków na refund w `RemoveTrackInternal`/`RemoveTurnout` — to była drukarka pieniędzy. Każdy NOWY placement rozjazdopodobny (wzór: `PlaceTurnoutOnChain`) MUSI: pre-check `CanAfford` za guardem `!SuppressCharging` → silence undo → charge po sukcesie + 1 komenda → `Original*` metadata na encji (restore+undo). `TurnoutGroszy` mapuje „Krzyż"/„Crossover" PRZED checkami liczbowymi (inaczej „Krzyżowy R190" łapie „190"). Uwaga: `PlaceCrossoverOnChain(…, flip, divergeLeft)` vs `PlaceTurnoutOnChain(…, divergeLeft, flip)` — odwrócona kolejność argumentów. Testy: `DepotConstructionBillingPlayTests`.
